Virtual Reality Haptic Touch

Overview

At the Stanford HackOverflow Hackathon I created the ability to feel holograms and virtual worlds. After my team connected the Leap Motion Controller to finger motors and an arduino via wires and solder, I built a javascript program to sense whenever a hand is touching a virtual surface and to turn on the corresponding finger motor(s) to vibrate. In addition, I built a lifelike virtual world for haptic touch users to play with our device and feel virtual shapes.

How it Works

The hardest part about this project was figuring out how to literally connect software signals to hardware circuitry. After a few hours I stumbled upon Johnny-Five, a JavaScript library that can communicate with hardware via the Firmata protocol. Using this and Node.js, I treated the finger motors as if they were simple LED's and wrote a program that turned on the motors after it received packages of JSON. Finally, using leap.js I wrote a 3D virtual reality which shoots off JSON packages whenever a finger touches a virtual surface.

StuddyBuddy Android Application

Previous Next

Overview

Summer 2015, I participated in Google CodeU, a virtual code camp for college students in computer science. In addition to learning from a Google mentor and practicing technical interviews, through Google CodeU, I took Udacity's Android Development course and completed the first version of StuddyBuddy (The 2 double d's are intentional). StuddyBuddy is an app to help students find study groups and educational assistance fast. A student user can create a 'Study Session' with relevant info about what academic material they need help with. Then, the Study Session gets uploaded to a Parse Datastore so that any other user will automatically have the new session added to their phone. This way, users can find study partners and groups close by to help and to receive help from.

Oscilloscope

Previous Next

Overview

I LOVE hacking hardware at hackathons and in my spare time. Without the right tools for debugging however, even a simple set up could take twice as long to build. I reimagined the traditional oscilloscope for research so that I would no longer have to bike 1.5 miles across campus for an oscilloscope to tell me exactly what was happening with my circuit.

How it Works

The goal was to build a small, low cost device that could sit on my dorm desk. I hypothesized that if I connect an external circuit to the sound card and configure the appropriate software, an average laptop could re-purpose as an oscilloscope. For robustness, I built the oscilloscope to provide protection against destructive input overloads, sound card calibration, voltage amplification and input attenuation to accomodate 20 mV to 20 V without distortion. Check out the my circuit documentation above to see how I accomplished this using extremely affordable circuit components and a lot of design thinking.

Smart Lights

Overview

I finished this project in 2013 as a freshman in college. I was excited by the recently released Philips Hue Smart Lights, and decided to extend their functionality by making them responsive to voice commands through Siri.

How it Works

In order to create this project, I needed the help of the Hue API, the Siri Proxy created by The-Three-Little-Pigs project and a Raspberry Pi. Luckily, the SiriProxy plugin format was very straightforward. SiriProxy will iterate through regexs in each plugin and if it doesn't find a match, forward the request to the normal Apple server. I wrote a script of commands that runs on the raspberry pi to control the lights and then made some voice command regexs to implement them.

About a year ago Apple released the HomeKit, which allows also users to control connected devices using Siri. In the future, I look forward to recreating this project using HomeKit.
